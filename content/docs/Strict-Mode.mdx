> StrictMode is a React tool for identifying potential problems in code.

> It does not affect production behavior but helps detect unsafe side effects, incorrect hooks, and deprecated patterns during development.

- React **intentionally calls the component twice** when mounting in Strict Mode.
- Purpose â€” to check that:
    - side effects are not performed outside of `useEffect`,

    - there are no resource leaks during mounting/unmounting.
