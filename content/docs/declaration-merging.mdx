> Declaration merging is a TypeScript mechanism that allows combining several declarations with the same name into a single extended type. The compiler "merges" them, combining properties, methods, or function signatures into a unified definition.

---

### ðŸ§© Main Idea

- If **multiple declarations with the same identifier** exist in the code, TypeScript automatically merges them.
- This applies to:
    - **interfaces**
    - **namespaces**
    - **functions and classes**, if they are declared compatibly with a namespace.

---

### ðŸ§© Example: Merging Interfaces

```tsx
interface User {
  id: number;
}

interface User {
  name: string;
}

const person: User = {
  id: 1,
  name: "Anna",
};
```

- Both definitions are combined:
    
    ```tsx
    interface User {
      id: number;
      name: string;
    }
    ```
    

> This is convenient for extending library types without directly editing them.

### ðŸ§© Example: Namespace + Function/Class

- You can add properties to functions or classes via a namespace.

```tsx
function greet(name: string) {
  return `Hello, ${name}`;
}

namespace greet {
  export const version = "1.0";
}

console.log(greet.version); // âœ… "1.0"
```

- The namespace "merges" with the function, creating a **hybrid entity**.

- Declaration merging = **combining multiple definitions into one**.
- Most commonly used for:
