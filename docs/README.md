# Project Documentation

This folder contains all project documentation, architecture decisions, rules, and guidelines.

## Contents

- **[ARCHITECTURE.md](./ARCHITECTURE.md)** - Source of truth for architecture decisions, technical stack, and implementation strategy
- **[APP_DOCUMENTATION.md](./APP_DOCUMENTATION.md)** - Technical documentation of the current application structure and features

## Adding New Documentation

When adding new documentation files, follow these naming conventions:

- `ARCHITECTURE.md` - Architecture decisions and technical design
- `RULES.md` or `GUIDELINES.md` - Development rules and coding standards
- `DEPLOYMENT.md` - Deployment procedures and configuration
- `CONTRIBUTING.md` - Contribution guidelines (if open source)
- `CHANGELOG.md` - Version history and changes
- `API.md` - API documentation (if applicable)

## Structure

```
docs/
├── README.md              # This file
├── ARCHITECTURE.md        # Architecture and technical decisions
├── APP_DOCUMENTATION.md   # Application technical docs
└── [other-docs].md        # Additional documentation
```
